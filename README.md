# 🩺 SkinDiseaseCheck

An offline Android application that leverages **Deep Learning** and **TensorFlow Lite** to classify common skin diseases from images selected from the device gallery.

The application performs **entirely on-device inference**, eliminating the need for an internet connection while providing fast, privacy-friendly predictions. As of **v0.2.0**, the app also includes a multi-stage input validation pipeline that filters out non-skin images (faces, screenshots, documents) before classification, reducing false predictions.

> **Disclaimer**
> This project is developed for educational and research purposes only. Predictions generated by the application are **not** a substitute for professional medical diagnosis or treatment.

# 📖 Overview

SkinDiseaseCheck demonstrates the deployment of a Deep Learning image classification model on Android using TensorFlow Lite.

The application allows users to select an image from their gallery. The image passes through a **validation pipeline** (face detection → text/screenshot detection → skin-tone check) before being preprocessed and classified by a TensorFlow Lite model, which predicts the most probable skin disease class along with a confidence score.

Since all processing occurs on-device, no user images are uploaded or stored.

# 🆕 What's New in v0.2.0

Earlier versions (v0.1.0) ran every selected image directly through the classifier, regardless of what the image actually contained — this meant faces, screenshots, and unrelated photos could all receive a confident-sounding disease prediction, which was misleading.

v0.2.0 introduces a **gated detection pipeline** that runs before classification:

| Stage | Purpose | How it changes results |
|---|---|---|
| **Face detection** (ML Kit, on-device) | Detects portraits/selfies | Images containing a face are rejected with *"This looks like a face or portrait"* instead of receiving a disease prediction |
| **Text recognition / OCR** (ML Kit, on-device) | Detects screenshots, documents, UI mockups | Text-heavy images are rejected with *"This looks like a screenshot or document"* instead of being misclassified |
| **Skin-tone heuristic** (YCbCr color analysis) | Confirms the image contains skin-like pixels, including inflamed/reddened and scaly tones | Non-skin images (objects, scenery, etc.) are rejected with *"No skin detected"*; the tone range was also widened so inflamed/diseased skin is no longer incorrectly rejected |
| **Confidence thresholding** | Flags low-confidence model outputs | Predictions below the confidence threshold are shown as *"Uncertain result"* with the raw confidence %, instead of silently guessing a disease name |

**Net effect:** the app now says "this isn't a valid image to analyze" instead of confidently naming a disease for images it was never meant to classify. This significantly reduces misleading false-positive predictions, at the cost of occasionally rejecting a borderline image that a user believes should have passed (see [Limitations](#-current-limitations)).

### Other changes in v0.2.0
- 🎨 Redesigned UI with Material Design cards, image preview, and color-coded result states (teal = confident, amber = uncertain, gray = rejected/no skin)
- 🐛 **Fixed:** prediction text was invisible on devices with system dark mode enabled (dark text rendering on a dark card background)
- ⚠️ **Known issue:** on some Xiaomi/MIUI devices, system-level dark mode may still override the app's forced light theme in rare cases — a fix is planned for a future release

# 🧠 Model Summary

The deployed classification model is a custom Convolutional Neural Network trained for multi-class skin disease classification.

Key techniques include:

- Convolutional Layers
- Max Pooling
- Batch Normalization
- Dropout
- Softmax Classification

The trained model was converted into **TensorFlow Lite (.tflite)** format for efficient mobile deployment. Note that the CNN model itself is unchanged in v0.2.0 — the improvements in this release come from the new input validation pipeline that runs before the model, not from retraining.

For a complete explanation of the model architecture, training procedure, hyperparameter tuning, and technical decisions, see:

# 🔄 Application Workflow

```text
Gallery Image
      │
      ▼
Face Detection (on-device)
      │  reject if face found
      ▼
Text Recognition / OCR (on-device)
      │  reject if screenshot/document
      ▼
Skin-Tone Heuristic (YCbCr analysis)
      │  reject if no skin-like pixels
      ▼
Image Preprocessing (resize, normalize)
      │
      ▼
TensorFlow Lite Model
      │
      ▼
Probability Prediction
      │
      ▼
Confidence Thresholding
      │  low confidence → "Uncertain result"
      ▼
Predicted Skin Disease + Confidence %
```

# ⚠ Current Limitations

- Limited to ten disease categories
- Performance depends on image quality
- Sensitive to lighting variations
- Cannot detect multiple diseases simultaneously
- No lesion localization or segmentation
- The validation pipeline (face/text/skin-tone checks) is heuristic-based, not learned — it can occasionally reject a valid borderline image or, less commonly, let through an invalid one
- Known dark mode rendering issue remains on some Xiaomi/MIUI devices
- Intended for educational and research purposes only
